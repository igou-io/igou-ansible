# netbootxyz on rb5009 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up the official netbootxyz container on the MikroTik RB5009 using the RouterOS container package, replacing the existing TrueNAS-hosted netbootxyz container as the homelab's PXE server.

**Architecture:** One top-level playbook (`playbooks/routeros/deploy_netbootxyz.yml`) orchestrates six tagged stages, each implemented as an `import_tasks` file under `playbooks/routeros/tasks/`. Stages are idempotent — every "did it change?" decision is gated on a `print where ...` read of router state, never on the exit code of `add`. The container's image is built off-device (control-node `podman pull` + `podman save`), uploaded once via `net_put`, and imported via `/container/add file=...`.

**Tech Stack:** Ansible (`community.routeros.command`, `ansible.netcommon.net_put`, `ansible.builtin.*`), RouterOS 7 container package, podman on the control node, ghcr.io/netbootxyz/netbootxyz multi-arch image.

**Spec:** [`docs/superpowers/specs/2026-05-07-netbootxyz-rb5009-design.md`](../specs/2026-05-07-netbootxyz-rb5009-design.md)

## Task 3: Implement `preflight` stage

**Files:**
- Modify: `playbooks/routeros/tasks/netbootxyz_preflight.yml`

- [ ] **Step 1: Replace stub with real preflight tasks**

```yaml
---
# Pre-flight checks before any netbootxyz stage runs.
# Verifies device-mode allows containers, architecture matches the chosen
# image platform, free disk is sufficient, and (control node side) podman
# is on PATH.
- name: Read device-mode
  community.routeros.command:
    commands:
      - "/system device-mode print"
  register: _devmode
  changed_when: false

- name: Assert containers are allowed by device-mode
  ansible.builtin.assert:
    that:
      - "'container: yes' in _devmode.stdout[0]"
    fail_msg: >-
      Containers are disabled in /system device-mode on
      {{ inventory_hostname }}. Run on the device:
        /system/device-mode/update container=yes
      then press the physical reset button when prompted.
      This playbook will not change device-mode.

- name: Read system resources
  community.routeros.command:
    commands:
      - "/system resource print"
  register: _resource
  changed_when: false

- name: Map image platform to expected RouterOS architecture-name
  ansible.builtin.set_fact:
    _expected_arch_map:
      "linux/arm64": "arm64"
      "linux/amd64": "x86_64"
      "linux/arm/v7": "arm"

- name: Assert architecture matches netbootxyz_image_platform
  ansible.builtin.assert:
    that:
      - "('architecture-name: ' ~ _expected_arch_map[netbootxyz_image_platform]) in _resource.stdout[0]"
    fail_msg: >-
      netbootxyz_image_platform={{ netbootxyz_image_platform }} expects
      architecture-name={{ _expected_arch_map[netbootxyz_image_platform] }}
      but {{ inventory_hostname }} reports a different architecture.
      Inspect /system resource print and update netbootxyz_image_platform.

- name: Parse free-hdd-space line
  ansible.builtin.set_fact:
    _free_match: "{{ _resource.stdout[0] | regex_search('free-hdd-space:\\s+([0-9.]+)([KMG])iB', '\\1', '\\2') }}"

- name: Compute free space in MiB
  ansible.builtin.set_fact:
    _free_mib: >-
      {%- if _free_match[1] == 'G' -%}
        {{ (_free_match[0] | float) * 1024 }}
      {%- elif _free_match[1] == 'M' -%}
        {{ _free_match[0] | float }}
      {%- else -%}
        {{ (_free_match[0] | float) / 1024 }}
      {%- endif -%}

- name: Assert >= 600 MiB free for image import
  ansible.builtin.assert:
    that:
      - (_free_mib | float) >= 600
    fail_msg: >-
      Free disk space on {{ inventory_hostname }} is
      {{ _free_mib | float | round(1) }} MiB; need >= 600 MiB to import
      the netbootxyz image tar. Investigate stale files under
      {{ netbootxyz_root_dir }}/ or other large entries in /file.

- name: Verify podman is available on the control node
  ansible.builtin.command: podman --version
  register: _podman_check
  delegate_to: localhost
  changed_when: false
  tags: [image]
```

## Task 4: Implement `image` stage

**Files:**
- Modify: `playbooks/routeros/tasks/netbootxyz_image.yml`

- [ ] **Step 1: Replace stub with image build + upload tasks**

```yaml
---
# Build the netbootxyz image tar on the control node, then upload to the
# router if size differs from what's already on flash. Sets fact
# netbootxyz_image_changed for the container stage to consume.
- name: Ensure local image cache directory exists
  ansible.builtin.file:
    path: "{{ netbootxyz_local_image_dir }}"
    state: directory
    mode: "0755"
  delegate_to: localhost

- name: Pull netbootxyz image on control node
  ansible.builtin.command: >-
    podman pull --platform={{ netbootxyz_image_platform }} {{ netbootxyz_image }}
  register: _podman_pull
  delegate_to: localhost
  changed_when: "'Writing manifest' in _podman_pull.stdout or 'Copying blob' in _podman_pull.stdout"

- name: Save image to tar (only if pull was changed)
  ansible.builtin.command: >-
    podman save {{ netbootxyz_image }}
    -o {{ netbootxyz_local_image_dir }}/netbootxyz.tar
  delegate_to: localhost
  when: _podman_pull.changed
  changed_when: true

- name: Stat local image tar
  ansible.builtin.stat:
    path: "{{ netbootxyz_local_image_dir }}/netbootxyz.tar"
  register: _local_tar
  delegate_to: localhost

- name: Fail if local tar missing (run with --tags image at least once)
  ansible.builtin.assert:
    that:
      - _local_tar.stat.exists
    fail_msg: >-
      {{ netbootxyz_local_image_dir }}/netbootxyz.tar does not exist on
      the control node. Run the image stage with --tags image first.

- name: Count router-side image tars matching local size
  community.routeros.command:
    commands:
      - "/file print count-only where name=\"{{ netbootxyz_root_dir }}/image.tar\" and size={{ _local_tar.stat.size }}"
  register: _router_tar_match
  changed_when: false

- name: Decide whether to upload image tar
  ansible.builtin.set_fact:
    _need_upload: "{{ (_router_tar_match.stdout[0] | trim | int) == 0 }}"

- name: Upload image tar to router
  ansible.netcommon.net_put:
    src: "{{ netbootxyz_local_image_dir }}/netbootxyz.tar"
    dest: "{{ netbootxyz_root_dir }}/image.tar"
  when: _need_upload
  register: _net_put_result

- name: Set netbootxyz_image_changed fact
  ansible.builtin.set_fact:
    netbootxyz_image_changed: "{{ _need_upload | bool }}"
```

Note on the size check: RouterOS `/file print` reports `size: <bytes>` for each entry. Substring match against the local stat size is pragmatic and avoids parsing the multi-line print output more deeply. Re-runs without an upstream image change will skip the upload.

## Task 5: Implement `network` stage

**Files:**
- Modify: `playbooks/routeros/tasks/netbootxyz_network.yml`

- [ ] **Step 1: Replace stub with veth + bridge tasks**

```yaml
---
# Create the container's veth (which holds the LAN-facing IP) and attach
# it to the existing bridge. Sets fact netbootxyz_network_changed for the
# container stage to consume.
- name: Count veths with our name
  community.routeros.command:
    commands:
      - "/interface veth print count-only where name={{ netbootxyz_veth_name }}"
  register: _veth_name_count
  changed_when: false

- name: Count veths matching desired address and gateway
  community.routeros.command:
    commands:
      - >-
        /interface veth print count-only where
        name={{ netbootxyz_veth_name }} and
        address={{ netbootxyz_address }} and
        gateway={{ netbootxyz_gateway }}
  register: _veth_match_count
  changed_when: false

- name: Add veth interface
  community.routeros.command:
    commands:
      - >-
        /interface veth add
        name={{ netbootxyz_veth_name }}
        address={{ netbootxyz_address }}
        gateway={{ netbootxyz_gateway }}
  when: (_veth_name_count.stdout[0] | trim | int) == 0
  changed_when: true
  register: _veth_added

- name: Update veth address/gateway if drifted
  community.routeros.command:
    commands:
      - >-
        /interface veth set [find name={{ netbootxyz_veth_name }}]
        address={{ netbootxyz_address }}
        gateway={{ netbootxyz_gateway }}
  when:
    - (_veth_name_count.stdout[0] | trim | int) >= 1
    - (_veth_match_count.stdout[0] | trim | int) == 0
  changed_when: true
  register: _veth_updated

- name: Count bridge ports matching desired veth and bridge
  community.routeros.command:
    commands:
      - >-
        /interface bridge port print count-only where
        bridge={{ netbootxyz_bridge }} and
        interface={{ netbootxyz_veth_name }}
  register: _bridge_port_count
  changed_when: false

- name: Add veth as bridge port
  community.routeros.command:
    commands:
      - >-
        /interface bridge port add
        bridge={{ netbootxyz_bridge }}
        interface={{ netbootxyz_veth_name }}
  when: (_bridge_port_count.stdout[0] | trim | int) == 0
  changed_when: true
  register: _bridge_port_added

- name: Set netbootxyz_network_changed fact
  ansible.builtin.set_fact:
    netbootxyz_network_changed: >-
      {{
        (_veth_added.changed | default(false)) or
        (_veth_updated.changed | default(false)) or
        (_bridge_port_added.changed | default(false))
      }}
```

## Task 6: Implement `container` stage

**Files:**
- Modify: `playbooks/routeros/tasks/netbootxyz_container.yml`

- [ ] **Step 1: Replace stub with full container management**

```yaml
---
# Configure /container global tmpdir, ensure the named mount and env list
# exist, then add+start (or recreate) the container row. Honors
# netbootxyz_image_changed, netbootxyz_network_changed, and
# netbootxyz_force_restart facts/vars to decide on remove+re-add.
- name: Create empty placeholder file locally
  ansible.builtin.copy:
    content: ""
    dest: "/tmp/netbootxyz-placeholder"
    mode: "0644"
  delegate_to: localhost
  changed_when: false

- name: Materialize config directory on flash via placeholder write
  ansible.netcommon.net_put:
    src: "/tmp/netbootxyz-placeholder"
    dest: "{{ netbootxyz_root_dir }}/config/.placeholder"
  changed_when: false

- name: Read /container config
  community.routeros.command:
    commands:
      - "/container config print without-paging"
  register: _container_config
  changed_when: false

- name: Set /container tmpdir
  community.routeros.command:
    commands:
      - "/container config set tmpdir={{ netbootxyz_tmpdir }}"
  when: ('tmpdir: ' ~ netbootxyz_tmpdir) not in _container_config.stdout[0]
  changed_when: true

- name: Count container mounts with our name
  community.routeros.command:
    commands:
      - "/container mounts print count-only where name=netbootxyz-config"
  register: _mount_name_count
  changed_when: false

- name: Count container mounts matching desired src and dst
  community.routeros.command:
    commands:
      - >-
        /container mounts print count-only where
        name=netbootxyz-config and
        src={{ netbootxyz_root_dir }}/config and
        dst=/config
  register: _mount_match_count
  changed_when: false

- name: Add container mount
  community.routeros.command:
    commands:
      - >-
        /container mounts add
        name=netbootxyz-config
        src={{ netbootxyz_root_dir }}/config
        dst=/config
  when: (_mount_name_count.stdout[0] | trim | int) == 0
  changed_when: true

- name: Update container mount if src/dst drifted
  community.routeros.command:
    commands:
      - >-
        /container mounts set [find name=netbootxyz-config]
        src={{ netbootxyz_root_dir }}/config
        dst=/config
  when:
    - (_mount_name_count.stdout[0] | trim | int) >= 1
    - (_mount_match_count.stdout[0] | trim | int) == 0
  changed_when: true

- name: Probe each desired env entry
  community.routeros.command:
    commands:
      - >-
        /container envs print count-only where
        name=netbootxyz-env and
        key={{ item.key }} and
        value={{ item.value }}
  register: _env_probes
  changed_when: false
  loop: "{{ netbootxyz_env_extra }}"
  loop_control:
    label: "{{ item.key }}"

- name: Add env entries from netbootxyz_env_extra
  community.routeros.command:
    commands:
      - >-
        /container envs add
        name=netbootxyz-env
        key={{ item.item.key }}
        value={{ item.item.value }}
  loop: "{{ _env_probes.results }}"
  loop_control:
    label: "{{ item.item.key }}"
  when: (item.stdout[0] | trim | int) == 0
  changed_when: true

- name: Count container rows for our root-dir
  community.routeros.command:
    commands:
      - "/container print count-only where root-dir={{ netbootxyz_root_dir }}"
  register: _container_count
  changed_when: false

- name: Decide whether to recreate container
  ansible.builtin.set_fact:
    _container_present: "{{ (_container_count.stdout[0] | trim | int) >= 1 }}"
    _container_recreate: >-
      {{
        (netbootxyz_image_changed | default(false)) or
        (netbootxyz_network_changed | default(false)) or
        (netbootxyz_force_restart | bool)
      }}

- name: Stop container before recreate
  community.routeros.command:
    commands:
      - "/container stop [find root-dir={{ netbootxyz_root_dir }}]"
  when: _container_present and _container_recreate
  changed_when: true

- name: Wait for container to reach status=stopped before remove
  community.routeros.command:
    commands:
      - "/container print count-only where root-dir={{ netbootxyz_root_dir }} and status=stopped"
  register: _stop_check
  retries: 12
  delay: 5
  until: (_stop_check.stdout[0] | trim | int) >= 1
  changed_when: false
  when: _container_present and _container_recreate

- name: Remove container before recreate
  community.routeros.command:
    commands:
      - "/container remove [find root-dir={{ netbootxyz_root_dir }}]"
  when: _container_present and _container_recreate
  changed_when: true

- name: Add container row (fresh or recreate)
  community.routeros.command:
    commands:
      - >-
        /container add
        file={{ netbootxyz_root_dir }}/image.tar
        interface={{ netbootxyz_veth_name }}
        root-dir={{ netbootxyz_root_dir }}
        mounts=netbootxyz-config
        envlist=netbootxyz-env
        hostname={{ netbootxyz_container_hostname }}
        logging={{ 'yes' if netbootxyz_container_logging else 'no' }}
  when: (not _container_present) or _container_recreate
  changed_when: true

- name: Wait for container to leave status=extracting
  community.routeros.command:
    commands:
      - "/container print count-only where root-dir={{ netbootxyz_root_dir }} and status=extracting"
  register: _extract_check
  retries: 24
  delay: 5
  until: (_extract_check.stdout[0] | trim | int) == 0
  changed_when: false

- name: Count container rows currently running
  community.routeros.command:
    commands:
      - "/container print count-only where root-dir={{ netbootxyz_root_dir }} and status=running"
  register: _running_check
  changed_when: false

- name: Start container
  community.routeros.command:
    commands:
      - "/container start [find root-dir={{ netbootxyz_root_dir }}]"
  when: (_running_check.stdout[0] | trim | int) == 0
  changed_when: true

- name: Wait for container status=running
  community.routeros.command:
    commands:
      - "/container print count-only where root-dir={{ netbootxyz_root_dir }} and status=running"
  register: _running_wait
  retries: 12
  delay: 5
  until: (_running_wait.stdout[0] | trim | int) >= 1
  changed_when: false
```

Note on the placeholder write: RouterOS doesn't have a `mkdir` primitive over network_cli. Writing an empty file at `<root-dir>/config/.placeholder` causes RouterOS to materialize the directory. The mount add then succeeds. `changed_when: false` keeps recap noise down.

## Task 7: Implement `menus` stage

- [ ] **Step 1: Replace stub with menu push tasks**

```yaml
---
# Push iPXE menus from netbootxyz_menu_source_dir to the container's
# config volume on the router. Pushes both the canonical path
# (config/menus/) and the local mirror (config/menus/local/) — the
# mirror is the existing TrueNAS workaround for netbootxyz overwriting
# config/menus/ on container restart unless MENU_VERSION=local.
- name: Enumerate local menu files
  ansible.builtin.find:
    paths: "{{ netbootxyz_menu_source_dir }}"
    file_type: file
    recurse: true
  register: _menu_files
  delegate_to: localhost

- name: Fail if menu source directory is empty
  ansible.builtin.assert:
    that:
      - _menu_files.matched | int > 0
    fail_msg: >-
      No files found under {{ netbootxyz_menu_source_dir }}.
      Verify the path or override netbootxyz_menu_source_dir.

- name: Probe each menu file at config/menus/<rel>
  community.routeros.command:
    commands:
      - >-
        /file print count-only where
        name="{{ netbootxyz_root_dir }}/config/menus/{{ item.path | relpath(netbootxyz_menu_source_dir) }}" and
        size={{ item.size }}
  register: _menu_canon_probes
  changed_when: false
  loop: "{{ _menu_files.files }}"
  loop_control:
    label: "{{ item.path | relpath(netbootxyz_menu_source_dir) }}"

- name: Push menus to config/menus/
  ansible.netcommon.net_put:
    src: "{{ item.item.path }}"
    dest: "{{ netbootxyz_root_dir }}/config/menus/{{ item.item.path | relpath(netbootxyz_menu_source_dir) }}"
  loop: "{{ _menu_canon_probes.results }}"
  loop_control:
    label: "{{ item.item.path | relpath(netbootxyz_menu_source_dir) }}"
  when: (item.stdout[0] | trim | int) == 0

- name: Probe each menu file at config/menus/local/<rel>
  community.routeros.command:
    commands:
      - >-
        /file print count-only where
        name="{{ netbootxyz_root_dir }}/config/menus/local/{{ item.path | relpath(netbootxyz_menu_source_dir) }}" and
        size={{ item.size }}
  register: _menu_local_probes
  changed_when: false
  loop: "{{ _menu_files.files }}"
  loop_control:
    label: "{{ item.path | relpath(netbootxyz_menu_source_dir) }}"

- name: Push menus to config/menus/local/ (TrueNAS workaround mirror)
  ansible.netcommon.net_put:
    src: "{{ item.item.path }}"
    dest: "{{ netbootxyz_root_dir }}/config/menus/local/{{ item.item.path | relpath(netbootxyz_menu_source_dir) }}"
  loop: "{{ _menu_local_probes.results }}"
  loop_control:
    label: "{{ item.item.path | relpath(netbootxyz_menu_source_dir) }}"
  when: (item.stdout[0] | trim | int) == 0
```

Note on the size match: each menu file is probed via a per-file `/file print count-only where name=... and size=...` query. Returns `1` if the file exists at the exact size, `0` otherwise. `net_put` runs only when the count is `0`. Same pattern as the existing `smoketest.yml`. Each loop iteration costs one round-trip to the router; for ~10 menu files × 2 passes that's ~20 trips per stage run, comfortably under a minute.
